Act as a senior backend architect and build a **production-ready, scalable backend API** using FastAPI and MySQL for a **Tea Vending Machine ERP System**.

Follow industry best practices, modular architecture, reusable components, and clean coding standards.

Focus only on Backend API and Database.

---

TECH STACK:

Backend Framework: FastAPI
Database: MySQL
ORM: SQLAlchemy
Schema Validation: Pydantic
Authentication: JWT-based authentication
Password Hashing: bcrypt
Email Service: SMTP
Background Tasks: FastAPI BackgroundTasks
Migrations: Alembic
Logging: Python logging module
Environment Variables: .env
Pagination: Limit/Offset
Soft Delete: is_active field

NAMING CONVENTION (VERY IMPORTANT):

All master tables must start with:

master_

Examples:

master_roles
master_users
master_leads
master_clients
master_items
master_machines
master_suppliers

Transaction tables must NOT use master_ prefix.

Every table must include:

id (Primary Key)
created_at (DateTime)
updated_at (DateTime)
is_active (Boolean default=True)
